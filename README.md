# Zubritunnel — точечный VPN для VSCode и Git Bash

Локальный HTTP-прокси, который заворачивает трафик **только выбранных приложений** через Outline / Shadowsocks-сервер с поддержкой `prefix` (TLS-обфускация для обхода DPI). Системный трафик не трогается.

С версии 2.0 поддерживается **несколько параллельных прокси** — каждый VPN-ключ можно поднять на своём порту (8080, 8081, 8082…) и одновременно гонять разные приложения через разные регионы.

## Быстрый старт

1. Распакуй папку куда удобно.
2. **Windows** — запусти `windows\install_shortcut.ps1` правой кнопкой → *Run with PowerShell*. На рабочем столе появится ярлык **Zubritunnel** с иконкой → двойной клик.
3. **Mac** — открой папку `mac/` в Finder, дважды кликни по `Zubritunnel.app`.
4. В окне жми **`+ ssconf://`** и вставь ссылку, которую дал провайдер VPN (формат `ssconf://...`). Ключ сохранится в `keys/`, появится в списке.
5. Выдели ключ → **Подключить** → **Тест ключа** (увидишь exit-IP).
6. Кнопками `VSCode` / `Git Bash` / `Chrome` / … запускай нужное — оно стартует через прокси выделенного ключа.

Системный трафик при этом идёт мимо VPN — провайдер видит только что ты подключён к серверу, не куда конкретно ходит браузер/мессенджер.

## Требования

| | GUI | CLI (только бинарник) |
|---|---|---|
| **Windows** | Python 3 ([python.org](https://www.python.org/downloads/), при установке поставь галку «Add Python to PATH») | ничего, .exe готов |
| **Mac** | Python 3 (есть в системе) + Go (`brew install go`) | Go |

GUI оборачивает Go-бинарник, поэтому Go всё равно нужен на Mac. На Windows бинарник уже собран.

## Что есть в окне GUI

- **Список ключей** — всё из `keys/`. Кнопки: `+ ssconf://`, `+ JSON`, `+ файл…`, **клонировать** (создаёт копию `<имя>-2.json` — удобно для одного ключа в разных регионах), **удалить**, **открыть папку keys/**
- **Подключить / Отключить** — поднимает/гасит прокси для выделенного ключа. Каждый ключ получает **свой порт**: первый 8080, второй 8081 и так далее. Зелёная точка слева от имени = ключ сейчас подключён
- **Несколько прокси одновременно** — можно подключить хоть пять ключей сразу, у каждого свой порт. Кнопки «Запустить через прокси» используют тот ключ, что выделен в списке
- **Свободный** — ручной перевыбор свободного порта (если 8080 уже занят чем-то, GUI сам подхватит 8081 — но если нужно «перепроверить», жми эту кнопку)
- **Сменить регион** — для ключей VanyaVPN и совместимых: тянет список доступных стран из API провайдера и одним кликом переключает сервер. Сама `ssconf://`-ссылка остаётся той же — провайдер на бэке меняет, на какой сервер она указывает
- **Двойной клик по ключу** — открывает окошко **«приложения на этом VPN»**: список процессов, запущенных через прокси этого ключа (PID и время старта)
- **Тест ключа** — проверяет, что ключ живой, показывает exit-IP и латентность
- **Проверить IP** — текущий IP через активный прокси
- **Запустить через прокси** — VSCode, Git Bash, PowerShell, Chrome, Firefox, Edge, Brave, Opera, Vivaldi, Yandex, Cursor, Terminal/iTerm — всё, что нашлось в системе. Плюс `Custom…` для произвольного `.exe` / `.app`
- **git config (вкл./выкл.)** — одна кнопка ставит `git config --global http.proxy` на порт выделенного ключа, другая снимает
- **Лог** — что происходит, ошибки

Тема тёмная, акцент — бирюза (#26C6DA, под цвет иконки V), шрифт Segoe UI, ttk-тема `clam`.

## Установка ярлыка / приложения

### Windows
Правой кнопкой по `windows\install_shortcut.ps1` → **Run with PowerShell**. Если ругается на ExecutionPolicy:
```powershell
powershell -ExecutionPolicy Bypass -File <путь>\windows\install_shortcut.ps1
```
В конце вылезет окошко «установка завершена», на рабочем столе появится ярлык **Zubritunnel**.

### Mac
Перетащи `Zubritunnel.app` в **Dock** для быстрого доступа (alias останется, .app живёт в `mac/`). В `/Applications` тоже можно — но **папка `mac/` должна остаться на месте**, лаунчер ищет `gui.py` относительно `.app`. Хочешь полностью переместить — переноси всю папку `mac/` целиком.

При первом запуске macOS может ругаться на «приложение от неопознанного разработчика» — System Settings → Privacy & Security → пункт «Zubritunnel was blocked» → **Open Anyway**.

## Альтернативные способы добавить ключ

**Вариант 1 — кнопка `+ JSON`:**
```json
{
  "method": "chacha20-ietf-poly1305",
  "password": "<password>",
  "server": "<host_or_ip>",
  "server_port": 443,
  "tag": "<регион / название>",
  "prefix": ""
}
```

**Вариант 2 — положи `.json` руками:**
Скопируй `keys/example.json.template` в `keys/<любое_имя>.json` и заполни поля. После следующего запуска ключ появится в списке.

**Вариант 3 — CLI:**
```bash
./vpn-proxy.exe -ssconf "ssconf://provider.example/path/uuid"
```

## Браузеры запускаются с изолированным профилем

Chrome / Edge / Brave / Opera / Vivaldi / Yandex / Firefox **не подхватывают переменные окружения** `HTTP_PROXY`/`HTTPS_PROXY` — они всегда лезут в свой основной профиль с собственными настройками. Поэтому Zubritunnel запускает их с отдельным профилем:

- Chromium-based: `--user-data-dir=%TEMP%\vpn-proxy-<brand>` + `--proxy-server=http://127.0.0.1:<порт>`
- Firefox: `-profile <временный>` + предустановленные `network.proxy.*`

Так браузер действительно ходит через VPN, а не делает вид. Закладок и истории основного профиля в этом окне не будет — это by design.

## CLI (без GUI)

### Windows
Двойной клик по `windows/run.bat` или из PowerShell:
```powershell
cd <путь>\windows
.\vpn-proxy.exe
```

### Mac
```bash
cd mac
chmod +x run.sh        # один раз
./run.sh
```

При первом запуске Go скачает зависимости и соберёт бинарник (~30 сек).

### Все флаги
```
-key <name>           выбрать ключ из keys/ по имени файла без .json
-keys-dir <path>      путь до папки с ключами (default: keys)
-config <path>        путь до отдельного JSON-файла
-ssconf <url>         скачать ключ напрямую из ssconf://
-addr 127.0.0.1:9090  сменить порт прокси
-launch code          запустить сразу VSCode без меню
-launch bash          запустить Git Bash (Win) / Terminal (Mac)
-launch "<path>"      запустить произвольную программу с прокси-env
-no-menu              просто держать прокси, ничего не запускать
```

Комбинации:
```bash
./vpn-proxy.exe -key <имя_ключа> -launch code
./vpn-proxy.exe -ssconf "ssconf://..." -no-menu
```

## Только git, без всего VSCode

Запусти прокси (пункт 4 в CLI-меню или `-no-menu`), затем в любом терминале:
```bash
git config --global http.proxy http://127.0.0.1:8080
git config --global https.proxy http://127.0.0.1:8080
```
Теперь `git push/pull/clone` идут через VPN, а `npm`, `curl`, браузер — мимо.

Снять:
```bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```

В GUI это делает кнопка **`git config (вкл./выкл.)`** — она автоматически подставит порт выделенного ключа.

## VSCode настройки

В `settings.json` (`Ctrl+,` → значок `{}` сверху-справа):
```json
{
  "http.proxy": "http://127.0.0.1:8080",
  "http.proxyStrictSSL": true
}
```
Если запускаешь VSCode через GUI-кнопку — это не нужно, прокси прокидывается через переменные среды. Замени `8080` на порт нужного ключа, если их несколько.

## Проверить, что VPN работает

```bash
# без прокси — твой реальный IP
curl https://ifconfig.me

# через прокси — IP VPN-сервера
curl -x http://127.0.0.1:8080 https://ifconfig.me
```
В GUI это делает кнопка **Проверить IP**.

## Если что-то сломалось

- **GUI не открывается из ярлыка/`gui.bat`** — открой PowerShell, `cd <путь>\windows`, `python gui.py` — увидишь traceback с ошибкой
- **Окно `run.bat` мерцает** — другой инстанс уже занял порт. В GUI порт подбирается автоматически; в CLI используй `-addr 127.0.0.1:9090` или жми **Свободный** в GUI
- **GUI висит на «connecting…»** — ключ битый или сервер недоступен. Жми **Тест ключа** для диагностики
- **Браузер открылся, но сайты не грузятся / показывает реальный IP** — ты запустил его не через GUI, а руками. Chromium/Firefox требуют отдельного профиля (см. раздел выше); проще нажать кнопку в Zubritunnel
- **«Сменить регион» ничего не показывает** — фича работает только для провайдеров, у которых есть публичный API списка стран (VanyaVPN и совместимые). Для обычных Outline-ключей кнопка серая
- **`vpn-proxy.exe` блокирует Windows Defender** — добавь папку в исключения (это open-source Go-бинарник, antivirus иногда триггерится на сетевые приложения)
